1) Start Here (Beginner How-To)

Goal: Get OpenClaw working in the simplest possible way, even if you're not technical.
  1. Install OpenClaw and open your terminal.
  2. Run openclaw gateway status. If not running, start/restart gateway.
  3. Add your chat token in config (Discord or Telegram).
  4. Set a simple allowlist policy so only trusted users can message.
  5. Send a test message and complete pairing when prompted.

Success check: You send a message and OpenClaw replies back in the same chat.

2) Quick Setup Checklist

  1. Install OpenClaw and confirm CLI health.
  2. Configure channel token(s) securely.
  3. Set DM/group policy explicitly (allowlist recommended).
  4. Start gateway and verify status.
  5. Complete pairing and test end-to-end reply.

3) Fast Triage Flow (60 seconds)

Is gateway running? → token valid? → pairing approved? → allowlist correct? → restart needed? → run doctor.

4) Top 10 Common Issues + Fixes

Discord DM not replying

Cause: Pairing not approved. Fix: Approve pairing and retest.

Reference: https://docs.openclaw.ai/channels/discord

Discord server messages ignored

Cause: Guild allowlist mismatch. Fix: Configure guild users allowlist explicitly.

Reference: https://docs.openclaw.ai/channels/discord

Telegram bot silent after setup

Cause: Token/gateway restart issue. Fix: Verify token and restart gateway.

Reference: https://docs.openclaw.ai/channels/telegram

Gateway fails after config edit

Cause: Schema validation error. Fix: Run doctor and fix invalid keys.

Reference: https://docs.openclaw.ai/gateway/configuration

Bot only responds on mention

Cause: requireMention behavior. Fix: adjust group mention config.

Reference: https://docs.openclaw.ai/channels/discord

6) Security Baseline

Mini Glossary (Beginner)

  • Gateway: the OpenClaw service process that handles messages.
  • Pairing: the one-time approval linking your account to the bot.
  • Allowlist: list of users/channels allowed to interact with the bot.
  • Token: secret key used to connect OpenClaw to a platform.
  • Use allowlist by default.
  • Rotate tokens if exposed.
  • Avoid public admin-surface exposure.
