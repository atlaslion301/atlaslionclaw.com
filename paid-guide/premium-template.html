1. The Autonomous Company Architecture

Stop running a single chatbot. To reach production scale, you need a multi-department company structure running inside your server. We use a strict 5-Stage Pipeline, where each department operates as an isolated subagent with a specific persona.

The Departments

  • The Scout (Opportunity): Trend Researcher. They scrape the web and write the Discovery Brief.
  • The Gatekeeper (Validation): Reality Checker. They verify technical feasibility before anyone writes a line of code.
  • The Forger (Build): Rapid Prototyper. They write the actual backend code and scripts.
  • The Megaphone (Growth): Growth Hacker. They write the marketing copy and viral hooks.
  • The Inspector (QA): Evidence Collector. They audit both the code from The Forger and the copy from The Megaphone.
  • The Architect (Scale): DevOps Automator. They deploy the finalized assets to AWS/Cloudflare.
Flow: Pipeline Handoff
Opportunity (Idea) ➔ Validation (Approval) ➔ Build/Growth (Execution) ➔ QA (Audit) ➔ Scale (Deployment)

2. Multi-Department SOP: The HQ Protocol

The Problem: Subagents go to sleep when they finish a task. Routing to a sleeping agent stalls the pipeline.

The Fix (The HQ Protocol): Never rely on a subagent to stay awake forever.

  1. Check subagent status via subagents list.
  2. Spawn a fresh worker for each task using sessions_spawn.
  3. Give them a strict 5-minute timeout window (timeoutSeconds: 300) so they have time to think.

3. Production JSON Configuration Templates

Here is the exact config.json block you need to securely run automated Cron jobs without exposing your main session.

{ "cron": { "enabled": true, "jobs": [ { "id": "daily-memory-compaction", "schedule": "0 20 * * *", "payload": { "kind": "systemEvent", "text": "Reminder: It's time for daily memory compaction." }, "sessionTarget": "main" } ] } }

4. Hardening & Security (P0 Guidelines)

  • Strict Allowlist: Never use "policy": "open". Always use "allowlist".
  • Execution Gating: Build and Growth must NEVER execute without a ✅ Pass from The Gatekeeper (Validation).
  • S3/CloudFront Delivery: Do not let agents serve files directly from the host filesystem. Push finalized assets to S3 and use Signed URLs for delivery.

5. Automated Recovery (Cron Backups)

Recovery Script: Daily Backup
Do not rely on memory alone. Use this exact cron expression inside your host server to backup your OpenClaw brain daily.
0 2 * * * tar -czf /backups/openclaw-$(date +\%F).tar.gz /home/user/.openclaw/workspace/
